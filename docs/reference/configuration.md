# Configuration reference

Every deployment is driven by a `config/inputs.<env>.yaml` file (generated by the wizard, then
yours to edit). This page documents the most-used keys. Re-running `Deploy-AKSLandingZone` after
editing this file is idempotent.

## Core identity & placement

| Key | Description | Example |
|---|---|---|
| `service_name` | Short workload name (3–10 lowercase chars). Drives resource naming. | `aksapplz` |
| `environment_name` | Environment token (≤8 lowercase alphanumeric). | `dev01` |
| `location` | Primary Azure region. | `swedencentral` |
| `secondary_location` | Set to enable a full second region. Leave empty for single-region. | `westeurope` |
| `topology` | `standalone`, `hub_and_spoke`, or `spoke`. | `standalone` |

## Cluster options

| Key | Default | Description |
|---|---|---|
| `kubernetes_version` | `1.33` | AKS control-plane version. |
| `sku_tier` | `Standard` | `Standard` or `Premium` (regulated scenarios require Premium). |
| `network_policy` | `calico` | `calico` or `azure` (Azure required for PCI segmentation). |
| `enable_workload_identity` | `true` | Entra Workload Identity. |
| `enable_azure_rbac` | `true` | Azure RBAC for Kubernetes authorization. |
| `enable_keda` | `true` | KEDA event-driven autoscaling. |
| `enable_vpa` | scenario | Vertical Pod Autoscaler. |
| `enable_node_auto_provisioning` | scenario | Karpenter-style node auto-provisioning. |

## Networking & ingress

| Key | Default | Description |
|---|---|---|
| `enable_app_gateway` | `true` | Application Gateway WAF v2 ingress. |
| `enable_agc` | `false` | Application Gateway for Containers (ALB): provisions the delegated subnet + NSG. |
| `enable_istio_service_mesh` | scenario | Istio service mesh with mTLS. |
| `global_lb_type` | `front_door` | Multi-region global LB: `front_door` or `traffic_manager`. |
| `subnet_address_prefix_*` | see below | Per-subnet CIDRs. |

### Default subnet layout (primary region)

| Subnet | CIDR |
|---|---|
| AKS system nodes | `10.10.0.0/24` |
| AKS user nodes | `10.10.1.0/24` |
| AKS API server | `10.10.2.0/24` |
| App Gateway | `10.10.21.0/24` |
| Private endpoints | `10.10.22.0/24` |
| Ingress / load balancer | `10.10.23.0/24` |
| App Gateway for Containers (AGC) | `10.10.24.0/24` |

The secondary region mirrors this under `10.20.x.0/24`.

## Supporting resources

| Key | Default | Description |
|---|---|---|
| `enable_managed_prometheus` | `true` | Azure Managed Prometheus. |
| `enable_managed_grafana` | `true` | Azure Managed Grafana. |
| `enable_defender` | `true` | Microsoft Defender for Containers. |
| `acr_geo_replication` | scenario | Geo-replicate ACR across regions. |
| `enable_backup` | scenario | Azure Backup for AKS. |

## GitHub & pipeline

| Key | Description |
|---|---|
| `github_organization_name` | The GitHub org that will own the workload repo. |
| `apply_approvers` | GitHub usernames who approve the `apply` environment. |
| `aks_admin_group_object_id` | Entra ID group granted cluster-admin via Azure RBAC. |
| `use_self_hosted_runners` | Provision an ACI self-hosted runner (default `false` → GitHub-hosted). |
| `template_repository` | `owner/name` of the repo hosting the reusable CI/CD workflows. |

!!! info "Full schema"
    The complete, authoritative list of keys lives in the
    [`config/inputs.yaml`](https://github.com/abengtss-max/aksapplz/blob/main/config/inputs.yaml)
    template and the Terraform `variables.tf` in the repo.
